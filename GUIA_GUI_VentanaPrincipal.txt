GUIA DETALLADA - GUI (VentanaPrincipal.cpp / VentanaPrincipal.h)
===============================================================

Esta guia esta enfocada SOLO en la parte grafica (la interfaz Qt):
- como se arma la ventana
- que widgets se crean
- como se organiza con layouts (diseno visual)
- como se actualiza la tabla y los labels
- como conectan los botones con la logica (via signals/slots) pero SIN entrar en el contenido interno de la matriz


0) Que archivos ver
---------------------
- include/GUI/VentanaPrincipal.h
- src/GUI/VentanaPrincipal.cpp
- src/Core/Aplicacion.cpp (solo para ver como se conecta la GUI)
- src/Core/main.cpp (para ver como se inicia la app)


1) Rol de VentanaPrincipal (vista)
----------------------------------
`VentanaPrincipal` hereda de `QMainWindow`, asi que crea una ventana principal con:
- un area central (central widget)
- una tabla `QTableWidget` para mostrar la matriz
- controles para definir un rango con `QSpinBox` (F1, C1, F2, C2)
- botones para disparar acciones
- labels para mostrar resultados

La GUI no calcula la suma/promedio/max/min. En su lugar:
- los botones disparan signals
- `Aplicacion` recibe esos signals y luego llama a metodos de la GUI para actualizar la tabla/labels


2) Estructura en VentanaPrincipal.h
-----------------------------------

2.1 Metodos (public)
- `inicializarTabla(int filas, int columnas)`
  Prepara la `QTableWidget`: tamano, encabezados y crea items en cada celda.
- `renderizarMatriz(const std::vector<CeldaInfo>& ocupadas)`
  Rellena la tabla: pone fondo blanco general y resalta solo las celdas ocupadas.
- `mostrarResultadoSuma(double suma)`
  Actualiza el label de suma.
- `mostrarResultadoRango(const std::vector<CeldaInfo>& rango)`
  Actualiza el label con el contenido del rango.

2.2 Signals (lo que emite la GUI)
La GUI define dos señales:
- `eventoSumarRango(int f1, int c1, int f2, int c2)`
- `eventoObtenerRango(int f1, int c1, int f2, int c2)`

Estas señales transportan los valores que el usuario eligio con los `QSpinBox`.

2.3 Slots (responden a eventos de la GUI)
Son privados y se conectan a los clicks de botones:
- `onBtnSumarClicked()`
- `onBtnObtenerClicked()`

2.4 Widgets miembro (lo que se guarda en la ventana)
- `QTableWidget* tablaMatriz`
- `QSpinBox* spinF1, *spinC1, *spinF2, *spinC2`
- `QPushButton* btnSumar, *btnObtener`
- `QLabel* lblResultadoSuma, *lblResultadoRango`


3) Como se construye la ventana (constructor + configurarUI)
------------------------------------------------------------

3.1 Constructor
En `VentanaPrincipal::VentanaPrincipal(QWidget *parent)`:
- llama a `configurarUI()`

O sea: el "diseno visual" se arma en `configurarUI()`.

3.2 configurarUI(): que se crea y como se organiza
Paso a paso (diseno):

1) Ventana principal
- `setWindowTitle("Visor de Matriz Dispersa")`
  Texto que aparece en la barra superior.
- `resize(800, 600)`
  Tamaque inicial.

2) Central widget + layout principal
- `QWidget* widgetCentral = new QWidget(this);`
- `setCentralWidget(widgetCentral);`
  Asi `QMainWindow` sabe donde poner el contenido principal.
- `QVBoxLayout* layoutPrincipal = new QVBoxLayout(widgetCentral);`
  Diseno vertical: todo lo que se agregue va de arriba hacia abajo.

3) Tabla de la matriz
- `tablaMatriz = new QTableWidget(this);`
- `tablaMatriz->setEditTriggers(QAbstractItemView::NoEditTriggers);`
  Importante para explicar: el usuario NO puede editar celdas, solo ver.
- `layoutPrincipal->addWidget(tablaMatriz);`
  La tabla queda arriba (antes de los controles).

4) Grupo de controles (Operaciones por Rango)
- `QGroupBox* gbControles = new QGroupBox("Operaciones por Rango", this);`
  Este `QGroupBox` pone un "recuerdo visual" con titulo.
- `QHBoxLayout* layoutControles = new QHBoxLayout(gbControles);`
  Diseno horizontal: los widgets quedan lado a lado.

Dentro:
- `spinF1`, `spinC1`, `spinF2`, `spinC2` son `QSpinBox`
  - cada uno usa `setPrefix("F1: ")` (y asi con C1, F2, C2)
  - el texto que ve el usuario queda tipo "F1: 0", "C1: 3", etc.
- Se agregan al layout con `layoutControles->addWidget(...)`.

Botones:
- `btnObtener = new QPushButton("Obtener Rango", this);`
- `btnSumar = new QPushButton("Sumar Rango", this);`
- Se agregan al layout con `addWidget`.

Luego:
- `layoutPrincipal->addWidget(gbControles);`

5) Grupo de resultados
- `QGroupBox* gbResultados = new QGroupBox("Resultados", this);`
- `QVBoxLayout* layoutResultados = new QVBoxLayout(gbResultados);`
  Resultados verticales.
- Labels:
  - `lblResultadoSuma = new QLabel("Suma: -", this);`
  - `lblResultadoRango = new QLabel("Valores en Rango: -", this);`
    y hace `setWordWrap(true)` para que no se salga del ancho.
- Los labels se agregan con `layoutResultados->addWidget(...)`.
- Finalmente `layoutPrincipal->addWidget(gbResultados);`

6) Conectar clicks a slots
Al final de `configurarUI()`:
- `connect(btnSumar, &QPushButton::clicked, this, &VentanaPrincipal::onBtnSumarClicked);`
- `connect(btnObtener, &QPushButton::clicked, this, &VentanaPrincipal::onBtnObtenerClicked);`

Esto es lo que hace que la interfaz reaccione cuando el usuario presiona botones.


4) Que hacen los slots de botones (actualizacion via signals)
---------------------------------------------------------------

4.1 onBtnSumarClicked()
Cuando el usuario presiona "Sumar Rango":
- lee los valores actuales en:
  - `spinF1->value()`, `spinC1->value()`, `spinF2->value()`, `spinC2->value()`
- emite:
  - `eventoSumarRango(f1, c1, f2, c2)`

O sea: la GUI solo "dispara" el evento con parametros del usuario.

4.2 onBtnObtenerClicked()
Es igual, pero emite:
- `eventoObtenerRango(f1, c1, f2, c2)`


5) Como se arma la tabla (inicializarTabla)
--------------------------------------------
`VentanaPrincipal::inicializarTabla(int filas, int columnas)`

Esta funcion prepara visualmente la tabla antes de mostrar datos:

1) Define tamano
- `tablaMatriz->setRowCount(filas);`
- `tablaMatriz->setColumnCount(columnas);`

2) Actualiza limites de los QSpinBox
Para que el usuario no elija un indice fuera de la matriz:
- maxima para filas:
  - `spinF1->setMaximum(filas > 0 ? filas - 1 : 0);`
  - `spinF2->setMaximum(filas > 0 ? filas - 1 : 0);`
- maxima para columnas:
  - `spinC1->setMaximum(columnas > 0 ? columnas - 1 : 0);`
  - `spinC2->setMaximum(columnas > 0 ? columnas - 1 : 0);`

3) Encabezados (labels de filas y columnas)
- construye `QStringList headersFila` y `headersCol`
- rellena con indices:
  - fila: 0,1,2,...,filas-1
  - columna: 0,1,2,...,columnas-1
- asigna:
  - `setVerticalHeaderLabels(headersFila)`
  - `setHorizontalHeaderLabels(headersCol)`

4) Crea items en todas las celdas
Esto es MUY importante para entender renderizado:
- hay dos bucles:
  - por `r` (0..filas-1)
  - por `c` (0..columnas-1)
- en cada (r,c) crea:
  - `QTableWidgetItem* item = new QTableWidgetItem("");`
  - `tablaMatriz->setItem(r, c, item);`

Asi, cuando luego `renderizarMatriz()` haga `tablaMatriz->item(r,c)`, encontrara el item (no sera null).


6) Como se muestran los datos en la tabla (renderizarMatriz)
-------------------------------------------------------------
`VentanaPrincipal::renderizarMatriz(const std::vector<CeldaInfo>& ocupadas)`

Esta funcion "redibuja" la tabla en dos etapas:

Etapa A: reset visual completo
Para cada celda:
- obtiene el item: `tablaMatriz->item(r, c)`
- si existe:
  - `item->setText("");`
  - `item->setBackground(QBrush(Qt::white));`

Resultado: toda la grilla queda limpia (fondo blanco y texto vacio).

Etapa B: resaltar solo celdas ocupadas
Para cada `CeldaInfo` en `ocupadas`:
- obtiene el item: `tablaMatriz->item(celda.fila, celda.columna)`
- si existe:
  - `item->setText(QString::fromStdString(celda.valor));`
  - `item->setBackground(QBrush(QColor(144, 238, 144)));`

El color `QColor(144, 238, 144)` es un verde claro.

Nota de explicacion para examen:
- la tabla se inicializa primero con items en `inicializarTabla`
- despues se pinta con `renderizarMatriz`


7) Como se muestran los resultados (labels)
---------------------------------------------

7.1 mostrarResultadoSuma(double suma)
- hace:
  - `lblResultadoSuma->setText("Suma: " + QString::number(suma));`

Entonces el label cambia de "Suma: -" a "Suma: X".

7.2 mostrarResultadoRango(const std::vector<CeldaInfo>& rango)
Construye un string:
- inicia: `QString resultado = "Valores en Rango: ";`
- si `rango.empty()`:
  - agrega "Ninguno"
- si no esta vacio:
  - recorre y agrega por cada celda:
    - `"[fila,col]=valor "` usando `arg(...)`
- al final:
  - `lblResultadoRango->setText(resultado);`


8) Como la GUI se conecta con el resto (Aplicacion y main)
-----------------------------------------------------------
Aunque pediste no entrar en la logica de la matriz, esto si es parte del "armado GUI"
(el puente entre ventana y acciones):

8.1 main.cpp (arranque)
- crea `QApplication app(argc, argv);`
- crea `Aplicacion aplicacion;`
- llama `aplicacion.iniciar();`
- entra al loop de eventos con `return app.exec();`

Ese loop de eventos es el que hace que la ventana responda a clicks.

8.2 Aplicacion.cpp: wiring de signals/slots
En el constructor `Aplicacion::Aplicacion(...)`:
- se crea:
  - `matriz = new MatrizDispersa(10, 10);`
  - `ventana = new VentanaPrincipal();`
- y conecta signals:
  - `connect(ventana, &VentanaPrincipal::eventoSumarRango, this, &Aplicacion::manejarSumarRango);`
  - `connect(ventana, &VentanaPrincipal::eventoObtenerRango, this, &Aplicacion::manejarObtenerRango);`

En `Aplicacion::iniciar()` (parte visual):
- pobla una matriz de prueba (esto alimenta datos a la GUI, no es diseno)
- llama el orden visual:
  1) `ventana->inicializarTabla(matriz->getFilas(), matriz->getColumnas());`
  2) `ventana->renderizarMatriz(matriz->listarOcupadas());`
  3) `ventana->show();`

Cuando el usuario hace click:
- VentanaPrincipal emite una signal con F1,C1,F2,C2
- Aplicacion recibe esa signal y luego llama:
  - `ventana->mostrarResultadoSuma(...)` o
  - `ventana->mostrarResultadoRango(...)`


9) Resumen para que lo puedas explicar facil (GUI)
----------------------------------------------------
Puedes decirlo asi en forma oral:

1) "La ventana es un `QMainWindow` con un `QTableWidget` en la parte superior."
2) "Abajo hay un `QGroupBox` con cuatro `QSpinBox` (F1,C1,F2,C2) y dos botones."
3) "Tambien hay un `QGroupBox` de resultados con dos labels."
4) "Primero la GUI crea la grilla completa en `inicializarTabla` (setRowCount/setColumnCount, headers y items)."
5) "Luego `renderizarMatriz` limpia todo la tabla a blanco y despues pinta solo las celdas ocupadas con texto y color verde."
6) "Los botones emiten signals; `Aplicacion` recibe esas señales y actualiza los labels con los resultados."


Fin de la guia.
